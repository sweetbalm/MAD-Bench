NGINX Official Recommended Configuration

High‑performance web server & reverse proxy — hardened settings based on F5 NGINX documentation

🟢 NGINX Open Source / NGINX Plus

Recommended version: 1.26.x (stable) / NGINX Plus R31+
Official Reference Architecture
⚙️ Critical Directives (nginx.conf)
  • worker_processes auto (equal to number of CPU cores). For high‑traffic, pin to cores: worker_processes 4;
  • worker_connections 1024 – 16384 per worker. Formula: max clients = worker_processes × worker_connections.
  • multi_accept on — accept as many connections as possible per epoll wakeup.
  • use epoll (Linux) or kqueue (BSD). Automatically selected but explicit is safe.
  • keepalive_timeout 65 – 120 seconds. Balance between resource usage and latency.
  • keepalive_requests 1000 (max requests per keep‑alive connection).
  • sendfile on — zero‑copy for static files, reduces CPU overhead.
  • tcp_nopush / tcp_nodelay on for both: optimizes packet transmission for static & dynamic content.
🚀 Proxy & Performance Tuning
  • client_max_body_size 10M – 100M (adjust to application requirements). Default 1M.
  • client_body_buffer_size 16k – 128k — avoid disk I/O for small requests.
  • proxy_buffering on (for upstream protection) or off for real‑time streaming.
  • proxy_buffer_size / proxy_buffers 16k 8 → proxy_buffer_size 16k; proxy_buffers 8 16k;
  • proxy_cache_path / proxy_cache Enable caching: proxy_cache_path /var/cache/nginx levels=1:2 keys_zone=static:10m inactive=24h max_size=5g;
  • gzip on with gzip_comp_level 4–6, gzip_types text/plain application/json etc.
  • ssl_session_cache / ssl_session_timeout shared:SSL:10m; timeout 10m — improves TLS handshake speed.
🖥️ System & Hardware Recommendations (Official NGINX Guidelines)
CPU
2+ cores (min), 8–16 cores for high concurrency
RAM
2GB – 32GB+ (depends on cache size & connections)
Storage
SSD / NVMe for static assets & proxy cache
Network
10GbE recommended for reverse proxy / load balancer
  • Linux Kernel Tuning Set net.core.somaxconn = 65535, net.ipv4.tcp_max_syn_backlog = 65535, net.core.netdev_max_backlog = 65535 in sysctl.conf.
  • File Descriptors Increase system limits: ulimit -n 65535 and worker_rlimit_nofile 65535 in nginx config.
  • Epoll / Event Model Ensure events { use epoll; worker_connections 16384; multi_accept on; }
# Example high‑performance configuration snippet (official style)
user nginx;
worker_processes auto;
worker_rlimit_nofile 65535;

events {
use epoll;
worker_connections 16384;
multi_accept on;
}

http {
sendfile on;
tcp_nopush on;
tcp_nodelay on;
keepalive_timeout 75s;
keepalive_requests 1200;
client_body_buffer_size 128k;
proxy_buffers 8 32k;
proxy_buffer_size 32k;
gzip on;
gzip_comp_level 5;
...
}
📘
Official recommendation: Always use NGINX Amplify or monitoring to observe metrics. Enable access_log with buffer=32k flush=5s for I/O reduction. For production reverse proxy, enforce TLS 1.3 and use HSTS preload. Configure upstream keepalive: keepalive 32 in upstream block to reduce connection buildup.
🔒
Security Hardening: Hide NGINX version (server_tokens off), restrict methods (limit_except), set appropriate timeouts (proxy_connect_timeout 5s; proxy_read_timeout 60s). Use allow/deny rules for admin panels.
